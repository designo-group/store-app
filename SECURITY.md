# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in **Støre**, please report it responsibly.

- Do **not** open a public GitHub issue for security vulnerabilities.
- Instead, please contact the maintainers directly at: [security@designo-group.com](mailto:security@designo-group.com)
- Provide as much detail as possible, including steps to reproduce the issue.

You will receive an acknowledgment as soon as possible, and we will work to resolve the issue promptly.

---

## Scope

This security policy applies only to vulnerabilities in the Støre application itself, including:

- The macOS user interface
- Internal logic and data handling
- Homebrew command execution integration
- Any code contained within this repository

---

## Homebrew Security Issues

Støre is a graphical interface built on top of Homebrew.

If your security concern relates to:

- Homebrew core functionality
- Formulae or casks
- Homebrew package definitions
- Homebrew installation scripts
- The Homebrew CLI itself

Please report those issues directly to the Homebrew team via their official channels:

https://github.com/Homebrew/brew

Støre does not maintain or modify Homebrew itself.

---

## Supported Versions

Only the latest released version of Støre is actively supported with security updates.
